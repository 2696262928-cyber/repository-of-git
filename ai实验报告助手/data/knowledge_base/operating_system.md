# 操作系统课程知识库

本文件用于“操作系统类实验报告”的 RAG 检索增强。内容参考 OSTEP、Linux man-pages 等公开资料，并结合进程、线程、调度、内存、文件系统实验改写为评分参考。它关注系统调用行为、并发正确性、资源管理和实验可复现性。

## 报告结构与评分关注点

操作系统实验报告应包含：实验目的、环境配置、相关原理、系统调用或内核机制、实验设计、核心代码、运行结果、现象解释、问题分析和总结。评阅时重点看“是否说明 OS 概念和实验现象的对应关系”“是否给出可复现命令”“是否分析进程/线程/内存/文件状态变化”“是否解释异常结果”。

高质量报告会写清楚操作系统版本、编译命令、运行命令、输入参数、观测工具和输出含义。例如进程实验应记录 PID、PPID、退出状态；线程实验应记录共享变量、锁和竞态结果；文件系统实验应记录文件描述符、权限、偏移量和缓冲行为。

## 进程、fork、exec、wait 与退出状态

进程实验常涉及创建、替换、等待和回收。Unix/Linux 中，fork 会创建子进程，父子进程从 fork 返回后的下一条语句继续执行，但返回值不同；exec 系列调用会用新程序映像替换当前进程；wait/waitpid 用于父进程等待子进程结束并获取退出状态。

评分时应检查：报告是否解释父子进程输出顺序不确定的原因；是否记录 PID/PPID；是否处理 fork 失败；是否区分 exec 前后代码是否继续执行；是否正确解析子进程退出状态。若实验只给输出截图而没有解释调度导致的顺序变化，应提示补充。

## 线程、互斥锁、条件变量与竞态

线程共享同一进程地址空间，因此并发访问共享变量时可能出现竞态条件。互斥锁用于保护临界区，条件变量用于让线程在某个谓词未满足时等待，并由其他线程在状态变化后通知。报告应说明共享资源、临界区、加锁范围和等待条件。

评阅时应关注：是否用实验对比“未加锁”和“加锁后”的结果；是否说明死锁、饥饿、优先级反转等风险；条件变量是否配合互斥锁和循环检查条件；是否避免在锁内执行耗时操作。若只写“加锁保证线程安全”但没有说明保护对象和竞态过程，应判为分析不足。

## CPU 调度与性能指标

调度实验常见算法包括 FCFS、SJF、优先级调度、时间片轮转、多级反馈队列等。报告应说明进程到达时间、服务时间、优先级、时间片、等待时间、周转时间、响应时间和吞吐量等指标。不同算法的目标不同，不能只用一个指标判断优劣。

评分时可检查：是否给出输入作业表；是否逐步展示调度顺序或甘特图；是否计算平均等待时间和平均周转时间；是否说明抢占式与非抢占式差异；是否分析时间片过大或过小的影响。若计算表和最终结论不一致，应提示复核公式。

## 内存管理、虚拟内存与页面置换

内存实验应说明地址空间、页、页表、缺页、页面置换和局部性。页面置换算法常见 FIFO、LRU、Clock 等。报告需要给出访问序列、物理块数量、每一步页面状态、缺页次数和缺页率。

评阅时应关注：是否区分逻辑地址和物理地址；是否展示页表或块状态变化；是否解释 Belady 异常或局部性影响；是否说明实验程序的内存访问模式。若只给一个缺页次数而没有过程表，难以判断算法实现正确性。

## 文件系统、I/O 与持久化

文件系统实验常涉及 open/read/write/close、文件描述符、权限、目录项、inode、缓冲区和同步写入。报告应说明文件打开模式、读写偏移、返回值检查、错误码和资源关闭。若实验涉及管道、重定向或进程通信，应说明数据流向和阻塞条件。

评分时应检查：是否处理系统调用失败；是否解释缓冲 I/O 与系统调用 I/O 的差异；是否说明文件权限和 umask；是否展示读写前后文件内容变化；是否避免遗漏 close 或资源泄漏。文件系统实验不能只展示最终文件，应说明操作过程。

## 死锁、同步问题与验证方法

死锁通常需要互斥、占有并等待、不可抢占、循环等待四个条件同时成立。报告应说明资源分配图或等待关系，并给出避免、预防或检测策略。生产者消费者、读者写者、哲学家进餐等同步问题应明确共享缓冲区、信号量/锁含义和线程执行顺序。

评阅时可关注：是否能复现死锁或竞态；是否提供日志显示线程状态；是否说明修复方案如何破坏死锁条件；是否验证修复后结果稳定。若报告只给理论定义而没有实验现象，应提示补充运行证据。

## 典型实验任务与检索关键词

常见操作系统实验包括 fork/exec/wait 进程创建、Shell 或命令解释器、管道通信、线程同步、生产者消费者、读者写者、调度算法模拟、页面置换算法、文件系统调用、死锁检测和内存分配。检索时可关注“进程、线程、PID、PPID、wait、互斥锁、条件变量、信号量、时间片、周转时间、缺页率、文件描述符、死锁”等关键词。

若报告出现“fork 输出顺序”，重点检查父子进程、调度不确定性和 wait。若出现“生产者消费者”，重点检查缓冲区、互斥和同步条件。若出现“LRU/FIFO/Clock”，重点检查访问序列和过程表。若出现“FCFS/SJF/RR”，重点检查甘特图和平均等待时间。若出现“管道、重定向”，重点检查文件描述符复制和关闭。

## 常见扣分点

- 没有操作系统版本、编译命令和运行命令。
- 只贴代码，没有解释系统调用返回值和状态变化。
- 进程输出顺序不确定却写成固定顺序。
- 并发实验没有对比未同步和同步后的结果。
- 调度、页面置换只给答案，不给过程表。
- 系统调用没有错误处理，忽略返回值。
- 总结没有结合实验现象，只写概念复述。

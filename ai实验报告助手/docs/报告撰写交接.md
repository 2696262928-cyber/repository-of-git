# 报告撰写交接

本文面向后续负责编写最终报告、录制演示视频和整理提交包的同学。

## 1. 项目一句话简介

项目名称：面向计算机类课程的实验报告智能检测与反馈系统。

系统支持学生上传实验报告，自动解析文档内容，结合课程 Rubric、本地 BM25 RAG 知识库和 DeepSeek/OpenAI-compatible 大模型，生成评分、问题建议、报告问答、修改计划或教师评语，并支持 DOCX/Markdown 导出、历史记录和用户反馈记录。

## 2. 核心亮点

最终报告中建议重点写以下亮点：

1. 多任务 Prompt 工程：全面评阅、按需求重点检查、基于报告问答、生成修改计划、生成教师评语。
2. 用户需求输入：用户可以在上传报告后输入自己的问题或关注点，使系统输出更贴合实际需求。
3. BM25 RAG + 来源引用：本地课程知识库覆盖 7 类计算机课程，并展示来源文件、章节、关键词和参考资料。
4. 结构化结果输出：要求大模型返回 JSON，便于前端展示和导出。
5. DOCX / Markdown 导出：评阅结果可以下载为文档，方便学生修改和教师存档。
6. 历史检测记录：保存最近检测摘要，方便回看不同样例和任务。
7. 用户反馈 / 学习记录：保存帮助度、采纳状态、薄弱点和后续修改计划，形成轻量学习闭环。
8. 增强测试样例：包含正常样例、缺陷样例和带图片 DOCX 样例。

## 8. 提交前注意事项

不要提交：

- `config/settings.json`
- `start_app_local.bat`
- `.venv/`
- `.pytest_cache/`
- `__pycache__/`
- `outputs/*`
- 含真实 API Key 的截图或日志

建议提交前检查：

```powershell
git status --short
.\.venv\Scripts\python.exe -m pytest -q
```

如果 `git status` 中出现 API Key、运行输出或缓存目录，应先确认 `.gitignore` 是否正确。

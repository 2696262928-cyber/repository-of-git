# 项目设计说明

## 一、项目名称

面向计算机类课程的实验报告智能检测与反馈系统

## 二、项目背景

计算机类课程通常包含大量实验环节，例如程序设计、数据结构、数据库、操作系统、计算机网络和人工智能实验。实验报告数量多、格式差异大、教师评阅工作量高，学生也常常不知道自己的报告是否完整、技术分析是否充分。

本项目希望利用公开大语言模型构建一个实验报告智能检测系统，为学生提供自查反馈，为教师提供评阅参考。

## 三、目标用户

- 学生：上传实验报告，获得结构完整性、技术内容和结果分析方面的修改建议。
- 教师：快速了解报告质量，获得评分参考和教师评语草稿。
- 助教：批量初筛报告，发现结构缺失、内容空泛和结果分析不足的问题。

## 四、核心功能

1. 实验报告上传与解析
2. 课程类型识别
3. 课程评分 Rubric 匹配
4. 实验报告智能检测
5. 结构化评分与问题反馈
6. 风险提示
7. 本地课程知识库检索增强
8. 图片与 OCR 辅助分析
9. 检测报告导出
10. 多任务 Prompt 与用户需求输入
11. 用户反馈与学习记录
12. 历史检测记录与结果追踪

## 五、系统流程

```text
用户上传报告
-> 文档解析模块提取文本、表格和代码
-> 文本凝练与图片分析流水线生成诊断信息
-> 课程分类模块识别课程类型
-> Rubric 模块读取评分标准
-> 本地知识库使用 BM25 按课程类型、任务类型和用户需求召回参考片段
-> LLM 根据任务类型选择 Prompt，生成评分、问答、修改计划或教师评语
-> 页面展示检测结果
-> 导出 Markdown / DOCX 检测报告
-> 保存历史检测摘要和用户反馈记录
```

## 六、模块说明

### 1. 文档解析模块

位置：`src/parser/`

负责解析 DOCX、PDF、TXT 和 Markdown 文件，提取报告正文、表格和代码块。

### 2. 课程分类模块

位置：`src/classifier/`

第一版采用关键词规则识别，后续可接入 LLM 分类 Prompt。

### 3. 评分规则模块

位置：`config/rubrics/` 和 `src/evaluator/rubric_loader.py`

不同课程对应不同 JSON 评分标准。

### 4. 大模型评阅模块

位置：`src/evaluator/llm_evaluator.py`

负责调用 OpenAI-compatible API，并根据任务类型选择不同 Prompt 模板。当前支持全面评阅、按需求重点检查、基于报告问答、生成修改计划和生成教师评语。不同任务使用不同 JSON 输出结构，系统会进行字段校验。

### 4.1 Prompt 工程模块

位置：`prompts/` 和 `docs/Prompt工程设计说明.md`

系统为不同任务设计独立 Prompt，包含角色设定、输入变量、输出格式、约束条件和 RAG 使用边界。用户输入的需求或问题会进入 Prompt，同时参与 RAG 检索，使模型输出更贴合用户关注点。

### 5. 本地知识库检索模块

位置：`src/retriever/` 和 `data/knowledge_base/`

系统根据课程类型优先读取对应 Markdown 知识文件，并基于报告关键词、当前任务类型和用户输入需求召回前若干片段。召回内容会显示在页面中，并在启用大模型评阅时追加到 Prompt，作为“本地课程知识库参考”。

当前版本采用轻量 BM25 相关度评分。知识库按 Markdown 二级标题切块，`参考来源` 章节不参与检索，而是作为元数据附加到召回片段中。页面、导出报告和 Prompt 会展示本地来源、章节标题、匹配关键词、BM25 分数和公开参考资料，便于说明 RAG 的依据。

知识库目前覆盖程序设计、数据结构、数据库、操作系统、计算机网络、人工智能与机器学习、嵌入式系统 7 类课程。每个课程文件包含报告结构、核心知识点、典型实验任务、检索关键词和常见扣分点。

### 6. 风险提示模块

位置：`src/evaluator/risk_detector.py`

提示结构缺失、引用缺失、内容过短、结果缺失等风险。

### 7. 图片与 OCR 分析模块

位置：`src/parser/ocr_parser.py` 和 `src/vision/`

系统支持 PDF/DOCX 图片 OCR、OpenCV 图片分类、图片质量检测、波形/表格/代码截图辅助识别，并将发现合并进预检诊断。

### 8. 结果导出模块

位置：`src/exporter/`

支持导出 Markdown 与 Word DOCX 检测报告，内容包括基本信息、总分、分项评分、问题建议、教师评语、风险提示和知识库引用。

### 9. 用户反馈与学习记录模块

位置：`src/utils/feedback.py` 和 `outputs/feedback.jsonl`

系统在检测结果后提供反馈表单，记录本次反馈是否有帮助、用户是否准备采纳、主要薄弱点和后续修改计划。该记录用于形成轻量学习闭环，帮助学生追踪报告修改重点。

### 10. 历史记录模块

位置：`src/utils/history.py` 和 `outputs/history.jsonl`

系统保存最近检测的文件名、课程类型、任务模式、用户需求、总分、问题数量、是否启用 LLM/RAG 以及 RAG 来源摘要，方便演示时回看不同任务的运行结果。`outputs/` 属于本地运行输出目录，不作为课程提交代码的一部分。

## 七、技术路线

```text
Streamlit + Python + 文档解析 + OpenCV/OCR + Rubric + BM25 RAG + 多任务 Prompt + DeepSeek/OpenAI-compatible LLM + Markdown/DOCX 导出 + 历史/反馈记录
```

推荐模型接入方式：

- DeepSeek API
- Qwen API
- 硅基流动 OpenAI-compatible API
- Ollama 本地模型

## 八、创新点

1. 面向多种计算机专业课程，而不是单一实验。
2. 将报告结构检测、技术正确性检测和教师评语生成结合。
3. 使用分课程 Rubric，提高评阅针对性。
4. 使用本地课程知识库检索增强，降低模型泛化评阅时的课程针对性不足问题。
5. 支持输出结构化评分、问题位置和修改建议。
6. 支持 Markdown / DOCX 双格式导出和历史检测记录。
7. 支持用户输入需求，使系统可以从“统一评阅”扩展到“按关注点检查、问答、修改计划、教师评语”等单轮任务。
8. 支持用户反馈和学习记录，形成“检测 -> 修改建议 -> 用户采纳反馈”的轻量闭环。
9. 风险提示采用辅助判断方式，避免绝对化判定。

## 九、后续扩展

- 自动提取代码并分析代码与结果一致性
- 增加批量检测
- 接入向量数据库或更专业的中文分词器，提升近义表达和跨章节语义召回能力
- 增加更细粒度的代码运行结果核验和图片证据定位

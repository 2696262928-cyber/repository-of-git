## 3. 实测记录填写建议

每次测试后建议补充：

- 是否能稳定完成检测。
- DeepSeek 是否返回合法 JSON。
- RAG 是否展示来源文件、章节、关键词和参考资料。
- 输出内容是否围绕用户需求。
- DOCX 是否能正常打开，表格和中文字体是否正常。
- 保存反馈后页面状态是否保留。

## 4. 已知边界

- 模型输出质量与 API 状态、网络和模型指令遵循能力有关。
- 规则模式只适合基础评分和风险提示，不适合完整问答或教师评语生成。
- RAG 使用 BM25，不是向量数据库，对近义词和隐含语义的召回能力有限。
- OCR 依赖图片清晰度，模糊截图、复杂背景和扫描件可能识别不完整。

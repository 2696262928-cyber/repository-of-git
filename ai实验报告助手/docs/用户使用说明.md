# 用户使用说明

本文面向老师、队友和普通使用者，说明如何启动系统、配置模型、上传报告、查看结果、导出报告和保存反馈。

## 1. 环境准备

推荐环境：

- Windows 10/11
- Python 3.10 或更高版本
- 浏览器：Edge、Chrome 或 Firefox

首次运行前，在项目目录执行：

```powershell
python setup_env.py
```

如果一键配置失败，可以手动执行：

```powershell
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
copy config\settings.example.json config\settings.json
```

项目目录指：

```text
D:\Projects\AI-Novices-Final-Project\_analysis_repository-of-git\ai实验报告助手
```

## 2. 启动系统

推荐方式：双击项目根目录下的：

```text
start_app.bat
```

命令行方式：

```powershell
cd D:\Projects\AI-Novices-Final-Project\_analysis_repository-of-git\ai实验报告助手
.\.venv\Scripts\streamlit.exe run app.py
```

本地开发方式：

```text
start_app_local.bat
```

`start_app_local.bat` 只用于本机开发，可能包含个人环境变量或 API Key，已被 `.gitignore` 排除，不应提交。

启动成功后，浏览器会打开：

```text
http://localhost:8501
```

如果 Streamlit 首次显示邮箱输入提示，直接留空回车即可。

## 3. 配置 DeepSeek API

系统不强制要求 API Key。未配置 Key 时，可以使用规则模式完成基础评分；配置 Key 后，可以启用多任务大模型生成。

配置文件：

```text
config/settings.json
```

示例：

```json
{
  "llm": {
    "provider": "openai_compatible",
    "base_url": "https://api.deepseek.com/v1",
    "api_key": "在这里填写你的 API Key",
    "model": "deepseek-chat"
  },
  "app": {
    "default_course_type": "auto",
    "max_report_chars": 20000,
    "output_dir": "outputs"
  }
}
```

也可以使用环境变量：

```powershell
$env:DEEPSEEK_API_KEY="你的 API Key"
.\.venv\Scripts\streamlit.exe run app.py
```

注意：不要把真实 API Key 写进代码、README、报告截图或 Git 提交记录。

## 4. 基本使用流程

1. 在左侧上传 DOCX / PDF / TXT / Markdown 实验报告。
2. 选择课程类型，或保持“自动识别”。
3. 在主页面“第二步：选择任务与输入需求”选择任务模式。
4. 输入自己的需求、问题或关注点。
5. 在左侧选择是否启用“大模型评阅”和“本地知识库 RAG”。
6. 点击“开始检测”。
7. 查看解析预览、RAG 引用、检测结果和原始 JSON。
8. 下载 Markdown 或 Word DOCX 检测报告。
9. 在“用户反馈 / 学习记录”中保存本次反馈是否有帮助、是否准备采纳和后续修改计划。

## 5. 任务模式说明

| 任务模式 | 用户需求是否必填 | 适用场景 |
| --- | --- | --- |
| 全面评阅 | 可选 | 按课程 Rubric 给出整体评分、问题和教师评语 |
| 按需求重点检查 | 必填 | 重点检查某个问题，例如“实验结果分析是否充分” |
| 基于报告问答 | 必填 | 针对报告提问，例如“事务实验设计是否充分？” |
| 生成修改计划 | 可选 | 输出按优先级排列的修改清单 |
| 生成教师评语 | 可选 | 生成教师批改风格的评语草稿 |

非全面评阅任务主要依赖大模型。如果关闭大模型，系统只能给出规则检测的基础结果，不能完整生成问答、修改计划或教师评语。

## 6. 查看解析预览

解析预览会展示：

- 文件名和文件类型
- 原始文本长度与凝练后文本长度
- 页数、图片数量、代码块数量
- 解析出的文本片段
- 图片分析和 OCR 结果

如果“凝练后”和“原始文本”长度接近，通常说明报告本身较短，系统没有必要压缩；这属于正常现象。

## 7. 查看 RAG 引用

启用“本地知识库 RAG”后，系统会从 `data/knowledge_base/` 检索课程知识片段，并展示：

- 本地来源文件
- 章节标题
- 匹配关键词
- BM25 分数
- 公开参考资料

RAG query 由以下内容组成：

```text
报告凝练文本 + 当前任务类型 + 用户需求
```

RAG 片段只作为课程背景和评阅依据，不代表学生报告中已经写了这些内容。

## 8. 下载 Markdown / DOCX

检测完成后，可以在导出区下载：

- Markdown 检测报告
- Word DOCX 检测报告

导出内容包括任务类型、用户需求、基础信息、评分/问答/修改计划/教师评语、问题建议、风险提示和 RAG 引用来源。

## 9. 保存用户反馈 / 学习记录

检测结果下方提供“用户反馈 / 学习记录”表单，可以记录：

- 本次反馈是否有帮助
- 是否准备采纳建议
- 主要薄弱点
- 备注 / 后续修改计划

记录保存到：

```text
outputs/feedback.jsonl
```

历史检测摘要保存到：

```text
outputs/history.jsonl
```

`outputs/` 是运行输出目录，已被 `.gitignore` 排除，默认不提交。

## 11. 常见问题

### 11.1 大模型调用失败怎么办？

先检查：

1. `config/settings.json` 是否存在。
2. API Key 是否正确。
3. `base_url` 是否是 `https://api.deepseek.com/v1`。
4. `model` 是否填写为 `deepseek-chat` 或平台支持的模型名。
5. 网络是否可访问模型接口。

即使大模型失败，系统仍可回退到规则检测模式。

### 11.2 为什么非全面评阅需要启用大模型？

“基于报告问答”“生成修改计划”“生成教师评语”等任务需要生成自然语言结构化结果，规则引擎只能提供基础评分和风险提示，不能替代这些生成能力。

### 11.3 为什么 feedback/history 不提交？

`outputs/feedback.jsonl` 和 `outputs/history.jsonl` 是本机运行记录，可能包含文件名、用户输入需求和评阅摘要，不适合作为代码提交内容。因此 `outputs/` 默认不提交。

### 11.4 DOCX/PDF/OCR 支持范围是什么？

- DOCX：支持正文、表格、代码块和嵌入图片解析。
- PDF：支持文本提取和图片辅助分析，扫描版 PDF 依赖 OCR 效果。
- OCR：使用 `rapidocr-onnxruntime` 做图片文字识别，适合清晰截图；模糊图片、复杂背景和手写内容识别效果有限。

### 11.5 RAG 为什么不用向量数据库？

当前版本使用轻量 BM25 检索，不需要额外部署向量数据库或 embedding 模型，便于课堂演示和本地运行。向量检索可以作为后续扩展方向。

# 测试用例

## 测试目标

验证系统能否完成以下任务：

1. 正确解析实验报告文本。
2. 正确识别课程类型。
3. 根据不同课程 Rubric 输出评分。
4. 给出具体问题和修改建议。
5. 验证多任务 Prompt、用户需求输入和 RAG 引用。
6. 导出 Markdown / DOCX 检测报告。
7. 保存历史检测记录和用户反馈 / 学习记录。

## 推荐测试样例

| 编号 | 课程类型 | 报告内容 | 预期检测重点 |
| --- | --- | --- | --- |
| T01 | 程序设计 | Python 基础语法或文件处理实验 | 代码说明、输入输出、运行结果 |
| T02 | 数据结构与算法 | 栈、队列、二叉树或排序实验 | 算法原理、复杂度、测试样例 |
| T03 | 数据库 | SQL 建表与查询实验 | 表结构、SQL 正确性、查询结果 |
| T04 | 操作系统 | 进程调度或线程同步实验 | 原理描述、运行过程、结果分析 |
| T05 | 计算机网络 | Wireshark 抓包实验 | 协议字段、抓包结果、网络参数 |
| T06 | 人工智能与机器学习 | KNN 或神经网络实验 | 数据集、模型、评价指标 |

## 缺陷样例设计

为了展示系统检测能力，可以准备几份存在明显问题的报告：

- 缺少实验结果
- 缺少实验总结
- 只有代码没有解释
- 运行结果与文字分析不一致
- 只有模板化文字，没有具体实验过程
- 缺少参考资料或数据来源

## 推荐手动测试组合

| 编号 | 输入文件 | 推荐任务模式 | 建议用户需求 | 预期检查点 |
| --- | --- | --- | --- | --- |
| M01 | `ai_knn_report.md` | 全面评阅 | 可不填 | 能识别 KNN、数据划分、标准化、K 值对比、混淆矩阵和误差分析 |
| M02 | `ai_knn_report.md` | 按需求重点检查 | 重点检查 KNN 参数选择和混淆矩阵分析是否充分 | 能围绕 K 值、标准化、混淆矩阵给出反馈 |
| M03 | `database_sql_report.md` | 基于报告问答 | 这份报告的事务实验设计是否充分？ | 能指出单会话回滚已验证，但并发隔离实验不足 |
| M04 | `bad_ai_only_accuracy_report.md` | 按需求重点检查 | 重点检查模型评估是否可靠 | 能指出训练集评估、数据泄漏、缺少测试集和混淆矩阵 |
| M05 | `bad_network_screenshot_only_report.md` | 生成修改计划 | 优先补充最影响得分的抓包证据 | 能指出缺少过滤条件、报文编号、Seq/Ack 字段解释和真实证据 |
| M06 | `programming_with_image_report.docx` | 全面评阅 | 检查图片结果是否能支撑报告结论 | 能解析 DOCX 图片对象，并给出报告完整性反馈 |
| M07 | `ai_knn_report.md` | 全面评阅 | 可不填 | 检测完成后能下载 Markdown / DOCX，并保存用户反馈记录 |

## 演示视频建议

演示时至少展示：

1. 上传一份完整报告。
2. 系统解析报告文本。
3. 系统识别课程类型。
4. 选择任务模式并输入用户需求。
5. 展示 RAG 引用来源。
6. 系统输出评分、问答、修改计划或教师评语。
7. 下载 Markdown / DOCX 检测报告。
8. 保存用户反馈 / 学习记录。
9. 展示一份缺陷报告的检测效果。

详细脚本见 `docs/演示视频脚本.md`。

# 演示视频脚本

建议视频时长：5-8 分钟。  
演示目标：让老师快速看到系统是一个完整的“AI + 实验报告评阅”应用，包含大模型生成、Prompt 工程、RAG 来源引用、可视化界面、导出和反馈记录。

## 0. 准备工作

录制前确认：

- 已能通过 `start_app.bat` 或命令行启动系统。
- `config/settings.json` 已配置可用 DeepSeek API Key，或环境变量已配置。
- 准备测试样例：
  - `data/samples/ai_knn_report.md`
  - `data/samples/database_sql_report.md`
  - `data/samples/bad_ai_only_accuracy_report.md`
  - `data/samples/bad_network_screenshot_only_report.md`
- 浏览器缩放建议 90%-100%，避免界面内容被遮挡。
- 桌面不要露出 API Key、私人文件或无关窗口。

## 1. 开场：项目背景（0:00-0:40）

讲述要点：

- 计算机类课程实验报告数量多、格式复杂。
- 学生需要知道报告结构是否完整、结果分析是否充分。
- 教师和助教需要快速获得评阅参考。
- 本项目使用大模型、Prompt 工程和本地课程知识库，构建实验报告智能检测与反馈系统。

推荐话术：

> 本项目面向计算机类课程实验报告场景，学生上传报告后，系统会解析文档、识别课程类型、结合 Rubric 和本地知识库，通过大模型生成评分、问题建议、修改计划或教师评语。

## 2. 系统界面与上传样例（0:40-1:30）

操作：

1. 打开 Streamlit 页面。
2. 展示左侧上传区域、课程选择、LLM/RAG 开关。
3. 上传 `data/samples/ai_knn_report.md`。
4. 课程类型选择“自动识别”或“人工智能与机器学习”。

讲述要点：

- 支持 DOCX / PDF / TXT / Markdown。
- 支持课程自动识别，也支持手动指定课程。
- 可启用 DeepSeek 大模型和本地知识库 RAG。

## 3. 多任务 Prompt 与用户需求（1:30-2:30）

操作：

1. 在“第二步：选择任务与输入需求”选择“按需求重点检查”。
2. 输入：

```text
重点检查 KNN 参数选择和混淆矩阵分析是否充分
```

3. 点击“开始检测”。

讲述要点：

- 系统不是单一聊天框，而是任务式 Prompt 工程。
- 支持全面评阅、重点检查、报告问答、修改计划、教师评语 5 种任务。
- 用户输入会同时进入 Prompt 和 RAG 检索 query。

## 4. RAG 引用展示（2:30-3:30）

操作：

1. 展开“本地课程知识库参考（RAG）”。
2. 展示来源文件、章节标题、匹配关键词、BM25 分数和参考资料。

讲述要点：

- 知识库覆盖程序设计、数据结构、数据库、操作系统、计算机网络、AI/机器学习、嵌入式 7 类课程。
- 当前使用轻量 BM25 检索，不需要向量数据库。
- RAG 片段只作为课程背景，不会被当作学生报告内容。

## 5. 检测结果展示（3:30-4:30）

操作：

1. 展示总分、问题建议、教师评语或专项检查结论。
2. 展示原始 JSON 折叠区。
3. 如时间允许，再用 `database_sql_report.md` 演示“基于报告问答”：

```text
这份报告的事务实验设计是否充分？
```

讲述要点：

- 结果是结构化 JSON，便于前端展示和导出。
- 模型输出会结合报告文本、Rubric、规则诊断和 RAG 引用。

## 6. 导出 DOCX / Markdown（4:30-5:20）

操作：

1. 点击下载 Markdown。
2. 点击下载 Word DOCX。
3. 打开 DOCX，展示标题、基本信息表、问题建议和 RAG 引用。

讲述要点：

- 导出内容可直接作为学生修改参考或教师批改记录。
- DOCX 使用正常中文字体和表格排版，便于提交和存档。

## 7. 用户反馈 / 学习记录（5:20-6:00）

操作：

1. 在检测结果下方填写反馈。
2. 选择是否有帮助、是否准备采纳。
3. 填写主要薄弱点和后续修改计划。
4. 点击保存。
5. 展示最近反馈记录。

讲述要点：

- 反馈记录形成轻量学习闭环。
- 学生可以追踪自己的薄弱点和修改计划。
- 运行记录保存在 `outputs/`，默认不提交，避免泄露私人数据。

## 8. 测试样例与局限（6:00-7:30）

操作：

1. 简要展示 `docs/测试用例.md` 或 `docs/测试结果记录.md`。
2. 展示缺陷样例 `bad_ai_only_accuracy_report.md` 或 `bad_network_screenshot_only_report.md` 的检测效果。

讲述要点：

- 自动测试当前覆盖解析、RAG、多任务 Prompt、导出、历史记录、反馈记录和样例质量。
- 缺陷样例用于验证系统能否发现缺少结果、缺少分析、评估不可靠、抓包证据不足等问题。
- 当前局限：不是企业级系统，RAG 不是向量数据库，OCR 受图片质量影响，模型输出依赖 API 稳定性。

## 9. 结尾（7:30-8:00）

推荐话术：

> 总体来看，本系统完成了从报告上传、文档解析、课程知识检索、大模型生成、结果展示、导出到用户反馈记录的完整流程，能够体现大模型协助下完成场景化教学应用系统的基本能力。
